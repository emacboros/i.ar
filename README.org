# i.ar

A self-modifying AI operating environment built in Emacs, running in a hardened Podman container, powered by local LLMs via Ollama. No cloud. No telemetry. No backdoors.

* Quickstart

** Prerequisites
- Podman (container runtime)
- Emacs 28+ (for local development without container)
- Ollama running on a reachable host (local or WireGuard mesh)
- A personalization repository (see below)

** Clone

#+begin_src bash
# Just want to use the tool (no personalization submodule):
git clone https://github.com/randazzo-ignacio/i.ar.git

# Want to work on i.ar or understand the codebase (includes personalization submodule):
git clone --recursive https://github.com/randazzo-ignacio/i.ar.git
#+end_src

The =personalization/= submodule contains knowledge bases, per-agent tasks, and audit logs. Users should create their own personalization repo with their own knowledge and agent configurations. See =personalization/= for the default setup.

** Build

#+begin_src bash
cd i.ar
./containers/build.sh
#+end_src

** Run

#+begin_src bash
# Basic usage (connects to remote Ollama via WireGuard):
./utils/emacboros.sh --personalization ~/repos/iar-personalization

# With self-modification enabled (for development/darwin):
./utils/emacboros.sh --personalization ~/repos/iar-personalization --self-modification

# With local Ollama:
./utils/emacboros.sh --personalization ~/repos/iar-personalization --local
#+end_src

** Inside Emacs

| Key | Action |
|-----|--------|
| C-c a | Load an agent (mirror, darwin, auditor, ctfwizard) |
| C-c k | Load a knowledge base (iar/, user/, infra/, etc.) |
| C-c p | Show prompt size (chars + approximate tokens) |
| C-c m | Summarize conversation to memory |

* Documentation

i.ar uses AI-first documentation. The knowledge base IS the documentation -- there is no separate wiki or architecture document.

To explore the project documentation interactively:

1. Load an agent (e.g., =C-c a mirror=)
2. Load the i.ar knowledge base (=C-c k iar/=)
3. Ask the agent about the architecture, design decisions, or code structure

The knowledge base files live in =personalization/knowledge/iar/= and cover:
- =architecture.md= -- Repository layout, container, network, security model
- =modules.md= -- Every Emacs Lisp module and what it does
- =agents.md= -- All agents, their roles, and the delegation system
- =tools.md= -- Every tool available to agents
- =philosophy.md= -- Design principles and decisions
- =usage.md= -- Detailed usage guide and personalization setup
- =workflow.md= -- Maintenance workflow (how to update docs after changes)
- =tool_gating.md= -- Security flags and power hierarchy

* Personalization

The =--personalization= flag mounts three subdirectories into the container:

- =knowledge/= -- Injectable knowledge bases (loaded via C-c k)
- =tasks/= -- Per-agent personal files (LOGS.md, SUMMARY.md, TODO.md, etc.)
- =audit/= -- Per-agent HISTORY.log files and global audit log

Create your own personalization repo with your own knowledge bases and agent configurations. Use the default =personalization/= submodule as a reference.
